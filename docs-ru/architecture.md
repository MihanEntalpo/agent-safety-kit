# Архитектура

`agsekit` построен вокруг workflow "хост плюс VM".

## Основные компоненты

- Хостовая машина: хранит реальный проект, каталог бэкапов и YAML-конфиг.
- VM Multipass: запускает Ubuntu и агентный процесс.
- Mount: показывает хостовую директорию внутри VM.
- Backup loop: делает снапшоты хостовой директории, пока работает агент.
- Port forwarding: держит SSH-туннели в живом состоянии, если они настроены.
- Опциональный proxy layer: применяет `proxychains` или `http_proxy` для установки и runtime.

## Поток выполнения

1. Пользователь описывает VM, mount и agents в YAML.
2. `agsekit` подготавливает хост и создаёт VM.
3. Хостовая директория монтируется в выбранную VM.
4. Бинарник агента устанавливается внутри VM.
5. `agsekit run` запускает агента внутри VM и при необходимости поднимает repeated backups.
6. При необходимости поверх этого включаются сетевые helper'ы.

## Граница изоляции

Хост хранит каноническое рабочее дерево и запускает VM. Агент не выполняется напрямую в хостовом shell. При этом mounted project всё равно остаётся writable из guest VM, поэтому сама VM не заменяет бэкапы и version control.

## Модель бэкапов

Бэкапы делаются на стороне хоста, а не внутри VM. Снапшоты создаются через `rsync` и hardlink'и, чтобы экономить место, когда меняется только часть дерева.

## Сетевая модель

- `proxychains` оборачивает команды, когда нужен SOCKS или proxy-aware runtime.
- `http_proxy` может быть direct или upstream через временный `privoxy`.
- `portforward` держит SSH forwards на основе конфига.
