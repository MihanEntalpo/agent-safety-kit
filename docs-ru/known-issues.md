## Текущие ограничения

- Есть проблемы с монтированием папок, в путях которых содержатся не-ascii-симолы, это проблема multipass.
- Есть проблемы с монтированием папок, в путях которых есть скрытые имена, это также проблема multipass.
- Если вы работаете несколькими пользователями на одном ПК, вам нужно иметь плюс-минус одинаковую конфигурацию agsekit, иначе возможны разнообразные проблемы
- Если вы работаете несколькими пользователями на одном ПК, никакой изоляции между пользователями не будет, так как multipass и её ВМ глобальны для всей машины, а не для пользователя.
- Пока нельзя автоматически изменить размер ВМ, после изменения её размеров в конфигурации, при необходимости это надо делать средствами multipass
- Windows host support пока не является first-class workflow. Используйте WSL.
- Linux-only интеграция с `systemd` пока не имеет нативной macOS-замены на `launchd`.
- Большие installers и сборки из исходников могут быть медленными на маленьких VM.
- Сам по себе agsekit run <agent> может быть не очень быстрым из-за подготовительных процедур и резервного копирования
- Системный демон пока работает только в linux на базе systemd

## Операционные оговорки

- конкретные proxy-настройки зависят от возможностей самого agent CLI (например claude-code не работает с proxychains)
- backup policy защищает файлы, но не защищает от внешних side effects вроде изменений в БД или сетевых действий
- если у вас в одной ВМ примонтировано несколько папок, то запуская агента в одной из них, вы всё равно рискуете остальными, хотя этот риск и значительно меньше, чем при запуске не хосте.
