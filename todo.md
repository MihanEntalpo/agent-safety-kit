## [v] Сделать блокировку резервных копий

* при начале резервных копий в конкретной папке, нужно создать pid-файл с номером процесса (файл создаётся в папке с бэкапами, напримeр backup.pid)
* если такой файл уже есть, и процесс жив, тогда нужно не делать резервные копии, а проверять раз в минуту, может быть процесс уже умер, или файл исчез
* если процесс умер, или файл исчез, тогда создаём свой файл, и начинаем делать резервные копии
* создание файла делаем с блокировкой файловой системы чтобы не могло быть состояние гонки
* в результате если запущено несколько агентов в одной и той же папке, резервные копии будут делаться только одним процессом, а не несколькими, при этом Если другие агенты завершатся, и их процессы резервного копирования прекратятся, резервным копированием займутся другие процессы, которые ещё работают

## [v] Реализовать более целостное отображение прогресса длинных операций

* Сейчас agsekit create-vms выполняет разные ansible playbook'и в цикле, и хотя каждый плейбук сам по себе отображает прогресс, общего прогресса не видно

* Нужно сделать некую систему "вложенных" прогрессов, когда например:
```
\ Установка виртуальных машин: Установка VM agents-personal
[##########_________] 50%
  | Установка VM: Установка бандлов
  [###############____] 73%
    - Установка Бандлов: installing docker
    [###________________] 50%
      / Установка docker: Gathering facts
      [#######____________] 43%
```
Здесь: 
* установка docker - это ansible-playbook установка бандла docker
* Установка бандлов - это цикл по всем заданным бандлам
* Установка VM - это набор всех действий по установке VM, среди который установка базовых пакетов, настройка ssh-ключей, и в том числе установка бандлов
* Установка виртуальных машин - это общий цикл установки на все имеющиеся VM'ы

В каждой строке после двоеточия указан текущий этап, то есть "Установка виртуальных машин" - это название блока, а "Установка VM agents-personal" - это его этап.

Названия этапов взяты для примера, значок "\", "|", "-", "/" в начале каждой строки - это крутилка, которая показывает прогресс

Надо также учесть что некоторые процессы могут идти параллельно, например 2 разные ВМ могут параллельно настраиваться, и тогда структура может выглядеть так:

```
\ Установка виртуальных машин:
[##########___________] 50%
  | Установка VM agents-personal: Установка бандлов           
  [###############____] 73%
    - Установка Бандлов: installing docker    
    [###______________] 50%
      / Установка docker: Gathering facts
      [#######________] 43%
  / Установка VM agents-work: Установка пакетов
  [####_______________] 43%
    - Установка пакетов: installing via apt-get
    [########_________]

```

Форматирование не обязательно именно такое.

Вроде бы есть готовые библиотеки по такого рода прогрессу с "вложенностью" этапов.
