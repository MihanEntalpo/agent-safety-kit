## Для агентов требующих авторизации через OAUTH сделать авто-проброс портов

* Сейчас, например, claude-code требует авторизации oauth, когда открывается ссылка, затем браузер после авторизации пытается открыть локальный порт, и не может 
* На этот случай нужна возможность автоматического проброса порта 
даже без специальной настройки через конфиг 
* Вероятно, понадобится какая-то специальная "изкоробочная" обработка таких вещей, специальная команда, вроде `agsekit auth-agent claude` которая будет знать что делать для каждого 
типа агента.

## Дублирующаяся установка агентов в ВМ

Если есть несколько агентов с одним и тем же типом, например:
```
agents:
  qwen-local:
    type:qwen

  qwen-cloud:
    type:qwen
```
тогда при установке agsekit install-agents --all-agents --all-vms агент qwen ставится по 2 раза.

Нужно чтобы ставились не сами агенты а их "типы"

## Реализовать команду update-vm

* Проверяет, если VM не того размера как указано в конфиге, спрашивает, надо ли изменить
* Изменяет через multipass (предварительно отключив)
* Если изменяется свободное место - сообщает о том что процесс будет долгим, и делает изменение размера, с увеличением места внутри ВМ 

## Реализовать команду agsekit up

* Запускает prepare, create-vms, update-vm, install-agents, и прочее
* Может быть запущено с опцией применяя только конкретные изменения
* Работает идемпотентно, может работать для обновления всего состояния в соответствии с конфигурацией
* Исправляет все точки монтирования, если их ВМ изменилась

## Реализовать команду agsekit down

* Отмонтирует все монтирования
* Выключает все VM

## Добавить в README.md (или в документацию) раздел known issues:

* Не желательно делать монтируемые папки со скрытыми путями (.name), их примонтировать не дадут настройки безопасности multipass
* Не желательно делать монтируемые папки с русскими (и другими UNICODE)-именами, с ними будут проблемы, связанные с тем, что SELinux видит эти папки иначе чем multipass (в виде набора escape-последовательностей)
* Есть проблемы при работе нескольких пользователей на одном компе:
  * multipass общий на всех и изоляции между пользователями нет и не будет
  * нужно иметь одинаковую конфигурацию, иначе будут проблемы между разными пользователями
