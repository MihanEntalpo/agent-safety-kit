# Документация
- При любом изменении `README.md` или `README-ru.md` нужно актуализировать второй файл с теми же сведениями на соответствующем языке.
- Если в задаче пользователя указано "увеличь версию", это означает, что нужно обновить версию в `pyproject.toml` и описать изменения в `changelog.md`.
- В `changelog.md` фиксируйте только изменения функционала приложения, без упоминания обновлений README.
- Перед выполнением любой задачи сверяйтесь с `SPEC.md`, чтобы учитывать актуальное описание системы.
- При изменениях архитектуры, команд CLI или логики системы обязательно обновляйте `SPEC.md`, чтобы он всегда соответствовал реализации.
- Перед запуском `pytest` нужно установить зависимости из `pyproject.toml` (например, через `pip install .` или `pip install -e .`).
- Всегда прогоняйте тесты после внесения изменений.
- Не запускайте автотесты для интеграции виртуальных машин в облачном окружении, но запускайте их в codex-cli.
