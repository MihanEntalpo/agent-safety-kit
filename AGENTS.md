# Project workflow guidance

В проекте обязательно должен быть файл `workflow.md`, в котором кратко описываются:

- все поставленные задачи, над которыми ведётся работа;
- проблемы и blockers, с которыми столкнулись по пути;
- способы, которыми эти проблемы были решены или планируется решить.

Перед началом работы нужно прочитать `workflow.md`, а при выполнении каждой новой задачи — обновлять его актуальной информацией.
